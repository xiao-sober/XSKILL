# Output Templates

Copy and adapt these templates when the user wants a stable structure.

## diary

```markdown
## 今日工作概述

[用 1-2 段概括今天的主线。说明目标、上下文和整体推进情况。]

## 主要完成内容

[按逻辑工作项整理，不按 commit 逐条罗列。说明具体做了什么，以及这些工作如何服务主目标。]

## 问题与处理过程

[描述遇到的问题、排查过程、关键判断、修复或缓解方式。没有明确问题时，可写“今天没有看到明确阻塞，主要是持续推进实现和整理。”]

## 当前结果或阶段性进展

[说明当前状态。区分“已完成”“阶段性完成”“正在排查”“仍需验证”。]

## 下一步计划

[列出 1-3 个自然的后续动作，只写由证据支持或用户已提到的计划。]
```

## report

```markdown
## 今日/本周完成事项

- [事项 1：动作 + 对象 + 结果/状态]
- [事项 2：动作 + 对象 + 结果/状态]

## 当前进度

[说明整体进展，避免夸大完成度。]

## 遇到的问题与处理情况

- [问题/风险点：排查或处理方式，当前状态]

## 风险或待确认事项

- [待确认项；如果没有明确风险，写“暂无明确阻塞，仍需继续验证......”]

## 下一步安排

- [下一步 1]
- [下一步 2]
```

## concise

```markdown
- 完成事项：[用一句或两句概括完成内容]
- 问题处理：[概括问题、排查、修复或当前阻塞]
- 下一步：[列出最直接的后续动作]
```

## Optional Evidence Note

```markdown
以下内容基于[提交记录/变更文件/用户备注/日志]整理，部分意图为保守推断。
```
